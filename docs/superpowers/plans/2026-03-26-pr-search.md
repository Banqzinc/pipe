# PR Search Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a search bar to the Inbox that filters PRs by title, author, branch name, or PR number.

**Architecture:** Backend adds a `search` query param to GET `/api/prs` using PostgreSQL `ILIKE` across title/author/branch_name/github_pr_number. Frontend adds a debounced search input to the Inbox header and passes the query through to all tab count queries.

**Tech Stack:** Express, TypeORM QueryBuilder, React, TanStack Query, Tailwind CSS

## Task 1: Backend — Add Search Query Param

**Files:**
- Modify: `src/routes/pr.routes.ts:25-44` (query param parsing + query builder)

- [ ] **Step 1: Add `search` to query params and build ILIKE filter**

In the GET `/api/prs` handler, the query params are destructured at line 25:

```typescript
const { status, repo_id, filter } = req.query as {
  status?: string;
  repo_id?: string;
  filter?: string;
};
```

Change to:

```typescript
const { status, repo_id, filter, search } = req.query as {
  status?: string;
  repo_id?: string;
  filter?: string;
  search?: string;
};
```

After the `repo_id` filter block (after line 42), add:

```typescript
if (search?.trim()) {
  const term = `%${search.trim()}%`;
  qb.andWhere(
    '(pr.title ILIKE :search OR pr.author ILIKE :search OR pr.branch_name ILIKE :search OR CAST(pr.github_pr_number AS TEXT) LIKE :search)',
    { search: term },
  );
}
```

This searches across title, author, branch name, and PR number. `ILIKE` is case-insensitive in PostgreSQL. `CAST` handles the integer PR number.

## Task 3: Frontend — Search Input in Inbox

**Files:**
- Modify: `frontend/src/routes/_authed/index.tsx` (state + UI)

- [ ] **Step 1: Add search state with debounce**

Add state near the other filter state:

```typescript
const [searchInput, setSearchInput] = useState('');
const [searchQuery, setSearchQuery] = useState('');
```

Add a debounce effect:

```typescript
useEffect(() => {
  const timer = setTimeout(() => setSearchQuery(searchInput), 300);
  return () => clearTimeout(timer);
}, [searchInput]);
```

Add `search` to the filters object that's passed to `usePullRequests`:

```typescript
const filters = {
  filter: activeTab === 'all' ? undefined : activeTab,
  repo_id: selectedRepo || undefined,
  search: searchQuery || undefined,
};
```

Note: The `search` param must be passed to ALL the `usePullRequests` calls (the ones for each tab count), not just the active tab query, so that counts reflect the search.

- [ ] **Step 2: Add search input UI**

In the Inbox header area (alongside "Sync PRs" button and repo select), add a search input. Use shadcn styling tokens:

```tsx
<input
  type="text"
  value={searchInput}
  onChange={(e) => setSearchInput(e.target.value)}
  placeholder="Search PRs..."
  className="bg-card border border-border rounded-lg px-3 py-1.5 text-sm text-foreground placeholder:text-muted-foreground focus:outline-none focus:ring-1 focus:ring-ring focus:border-transparent w-64"
/>
```

Place it between the page title and the Sync/repo controls for a natural layout.
